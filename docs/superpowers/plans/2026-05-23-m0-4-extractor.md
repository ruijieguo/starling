# M0.4 LLM Extractor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the LLM-driven Statement extractor as a transactional pipeline that turns one Engram into 0..N validated Statements with full ExtractionAttempt accounting and chunk-level idempotency, owning Bus.write(Statement) as the only sanctioned Statement persistence path.

**Architecture:** A `LLMAdapter` interface (P1: pluggable, P1 test ships a `FakeLLMAdapter` with deterministic XML fixtures) feeds raw Engram bytes plus an `existing_ref_map` short-id stub into an XML-strict parser. The parser produces `ExtractedStatement` POD records that the new `StatementWriter` (sole writer to `statements` + `bus_events` for `statement.written`) inserts under a single SQLite transaction guarded by `extraction_span_key` idempotency. The `Extractor` orchestrator opens a `PipelineRun(kind=extraction)`, walks chunks, retries on transient parse failure, and reports `Success / PartialSuccess / Failed` per-attempt to `extraction_attempt`. ConflictProbe stays out of scope (M0.5); Validator only enforces schema-shape + provenance/perspective/tenant_id rules.

**Tech Stack:**
- C++20 (extractor core, XML parser, StatementWriter)
- pybind11 bindings for the Extractor + LLMAdapter abstract base + ExtractedStatement
- Python `starling.extractor` package for prompt templates, adapter implementations, and integration glue
- SQLite 3.46+ (uses pre-existing `statements`, `pipeline_run`, `extraction_attempt`, `bus_events` tables — no new migrations)
- GoogleTest (C++) + pytest (Python) + ASan/UBSan
- LLM-adapter-agnostic: P1 ships FakeLLMAdapter only; OpenAI / Anthropic / local-compatible adapters land in P2

## Subsystem Contracts (locked at plan time)

### 1. extraction_span_key formula

```
extraction_span_key = sha256_hex(
    engram_ref.id           ⊕ \x1f ⊕
    str(chunk_index)        ⊕ \x1f ⊕
    predicate               ⊕ \x1f ⊕
    canonical_object_hash
)
```

`\x1f` = ASCII unit separator (the same delimiter used by `compute_idempotency_key` in `bus_event.cpp`). `predicate` is the Statement.predicate string (not URI-encoded). `canonical_object_hash` is what the M0.1 `canonicalize_object` returns — already a sha256 hex string, included verbatim. This is M0.4's choice to inline the hash rather than rehashing the canonical string; rationale: it keeps the key stable across canonicalize_object_hash_version bumps as long as the hash itself stays stable.

P1 normalized_span is `chunk_index` (per §5 of bus subsystem doc). When P2 lands `segment_map`, the formula extends to include `(segment_id, span_start, span_end)` — out of scope for M0.4.

### 2. ExtractedStatement POD

```cpp
struct ExtractedStatement {

    std::string                  holder_id;            // CognizerRef.id

    // content
    std::string                  subject_kind;         // "cognizer" | "entity"
```

### 4. PipelineLedger usage pattern

```
extractor.run(engram_ref, payload, holder_id, holder_tenant_id, existing_ref_map):
    run_id = ledger.start_run(tenant_id=holder_tenant_id,
                              input_ref=engram_ref.id,
                              metadata_json=json{prompt_version, extractor_version})
    emit pipeline.run_started (causation_parent = caller-supplied or evidence.appended)
    accepted_ids = []
    rejected_fragments = []
    for chunk_index, chunk_bytes in enumerate(chunked(payload)):  # P1: 1 chunk per Engram
        for attempt_no in 1..max_retries:                         # max_retries = 3
            response = adapter.extract(prompt, prompt_input_hash)
            if response.ok:
                parse_result = xml_parser.parse(response.raw_xml, existing_ref_map)
                if parse_result.errors:
                    if attempt_no < max_retries:
                        emit extraction.retry_scheduled
                        continue
                    rejected_fragments.append(...)
                    ledger.record_attempt(run_id, span_key, attempt_no, FAILED, ...)
                    emit extraction.failed
                    break
                # parse OK
                for stmt in parse_result.statements:
                    if not validator.validate(stmt):
                        rejected_fragments.append(...)
                        continue
                    span_key = compute_extraction_span_key(engram_ref, chunk_index, stmt.predicate, stmt.canonical_object_hash)
                    if extraction_span_key already in extraction_attempt with status=success:
                        ledger.record_attempt(run_id, span_key, attempt_no, status=NOOP, error=...)
                        emit extraction.noop
                        continue
                    write_outcome = bus.write(stmt, ..., causation_parent=run_started_event_id)
                    if write_outcome == ChunkDuplicate:
                        # the writer already forced REVIEW_REQUESTED; just record
                        accepted_ids.append(write_outcome.stmt_id)
                    else:
                        accepted_ids.append(write_outcome.stmt_id)
                ledger.record_attempt(run_id, span_key=chunk_span_key,
                                     attempt_no=attempt_no,
                                     status=SUCCESS,
                                     raw_output_hash=sha256(response.raw_xml))
                break
            else:
                if attempt_no < max_retries:
                    emit extraction.retry_scheduled
                    continue
                ledger.record_attempt(run_id, span_key=chunk_span_key,
                                     attempt_no=attempt_no,
                                     status=FAILED,
                                     error=response.error)
                emit extraction.failed
                break
    # all chunks done
    if accepted_ids and not rejected_fragments:
        ledger.finish_run(run_id, terminal=Finished)
        emit pipeline.run_completed
        return ExtractionRunResult(SUCCESS, accepted_ids, [])
    elif accepted_ids and rejected_fragments:
        ledger.finish_run(run_id, terminal=Finished)
        emit pipeline.run_completed   # PARTIAL is per-attempt, not per-run
        return ExtractionRunResult(PARTIAL_SUCCESS, accepted_ids, rejected_fragments)
    else:
        ledger.finish_run(run_id, terminal=Failed)
        emit pipeline.run_failed
        return ExtractionRunResult(FAILED, [], rejected_fragments)
```

The `for attempt_no in 1..max_retries` retry loop is per-chunk; the per-chunk `chunk_span_key` for ledger recording is a separate concept from the per-statement span_key for idempotency. The chunk_span_key is `sha256_hex(engram_ref.id ⊕ str(chunk_index))` — a coarser key — and represents one row per (chunk, attempt_no) in extraction_attempt. The per-statement span_keys are the §15.3.2 "幂等写入" guards used inside `Bus.write`.

### 5. holder_perspective rule table (verbatim into prompt)

| 场景 | holder_perspective |
|---|---|
| 主语为 self（agent 自己）且 present_in_episode | `first_person` |
| 主语为他人，原文是直接引语或明确转述 | `quoted` |
| 转述但不能追溯到原始来源（听说 / 传闻） | `hearsay` |
| LLM 推断生成的填空证据（无原文支持） | `inferred` |

Validator rejects empty / "default" / unrecognized values. Default-INFERRED bottom-out is forbidden.

### 6. confidence default rules (verbatim into validator)

| 来源 | 初值 |
|---|---|
| first-person 明确陈述 | ≥ 0.8 |
| 直接观察 | 0.7 - 0.9 |
| hearsay | 0.5 - 0.7 |
| 系统推断 | ≤ 0.5 |
| 任何 | < 0.3 → reject (validator drops, records as rejected_fragment) |
| 任何 | ≥ 0.3 但来自弱推断 | review_status = INFERRED_UNREVIEWED |

"弱推断" = `holder_perspective ∈ {INFERRED, HEARSAY}` OR `provenance == TOM_INFERRED` OR `confidence < 0.5`.

### 7. Validator rule set (M0.4 minimal — extends to §3.11 in M0.5+)

1. Required fields: `holder_id, holder_perspective, subject_kind, subject_id, predicate, object_kind, object_value, modality, polarity, confidence, observed_at, source_spans (≥1), provenance` — any missing → reject.
2. `tenant_id` derivation: `stmt.tenant_id = lookup_cognizer(holder_id).tenant_id`. P1 lookup is a stub map injected into Validator; M0.5 wires to a real Cognizer Hub. Stub returns "default" for any holder_id not in the map (matches the Cognizer M0.1 default-tenant rule).
3. `holder_perspective ∈ {first_person, quoted, hearsay, inferred}`. No defaults.
4. `polarity ∈ {pos, neg, unknown}`.
5. `modality` must be one of the 11 §3.4 values.
6. `object_kind` must be in {bool, int, float, str, datetime, cognizer, entity, statement}. Validator MUST refuse `list` / `dict` (P1 §3.3 Value rule).
7. `confidence ∈ [0.0, 1.0]`. `confidence < 0.3` → reject with reason "below_minimum_confidence".
8. Provenance × evidence/derived_from grid (M0.4 only emits `user_input` or `tom_inferred`):
   - `user_input` requires `source_spans ≥ 1`.
   - `tom_inferred` does not require `source_spans` (but may carry them as supporting). M0.4 leaves `derived_from` empty in all cases — derivation is M0.5+.
9. Cross-tenant rejection: if the parser outputs a Statement whose holder lives in a different tenant than the calling Bus.write request, reject with `extraction.failed(schema_invalid)` and record as rejected_fragment. The full §3.11 path (org_default / system / explicit-protocol) is M0.5+.

### 8. ExistingRefMap stub (M0.4 → M0.6 fillout)

```cpp
using ExistingRefMap = std::map<std::string, std::string>;  // short_id ("s1") -> StatementRef.id (UUID)
```

M0.4 always passes an empty map from the production code path. The XML parser still needs to handle `<statement_ref id="s1"/>` — when a short id is referenced and the map is empty, parser emits an `unresolved_short_id` error. This guarantees that nothing in production silently coerces unresolved short ids to fake long UUIDs.

When M0.6 lands the holder-aware retrieve, the orchestrator will populate the map by calling `basic_retrieve(holder=..., recent=true, limit=20)` and extracting candidate StatementRefs.

### 9. Bus.write transaction shape (M0.4 minimal)

`Bus.write` opens a `BEGIN IMMEDIATE / COMMIT` transaction, then under that single transaction:

1. `StatementWriter.write` — INSERT into `statements` (with the §15.3.2 chunk-duplicate detection happening before INSERT)
2. `OutboxWriter.append` — INSERT one `bus_events` row with `event_type=statement.written`

That's it for M0.4. `statement_edges` writes (for SUPERSEDES/CONFLICTS_WITH) are M0.5. Validator runs **before** the transaction starts; chunk-duplicate detection runs **inside** the transaction (so it's read-committed from the same transactional snapshot).

### 10. capability requirements (no new preflight ask)

M0.4 doesn't introduce a new ProfileCapability requirement. The existing `LOCAL_STORE_REQUIRED` tuple — including `cross_partition_transaction`, `transactional_outbox`, and `idx_statement_id_tenant` — is already the minimum that M0.4 needs (statements table + bus_events single-tx). Existing `relax_preflight_for_m0_3` continues to work as the M0.4 acceptance helper (rationale: real KMS is still M0.4+1, so the same `engram_per_record_key=False` relaxation applies). Reuse, don't rename.

### 11. Statement id generation

M0.4 reuses the same `random_id()` scheme as `OutboxWriter` and `PipelineLedger` (128 random bits with UUID-style dashes). This is documented as "pre-UUIDv7 placeholder". The eventual UUIDv7 swap is one PR away — they all read from the same helper which we'll lift into `include/starling/util/uuid.hpp` as part of this milestone (Task 1.5 in the build sequence).

### 12. retry / dead-letter policy (M0.4)

- `max_retries = 3` per chunk (extractor-internal constant; constant lives in `extractor.cpp` to avoid premature config surface).
- Transient parse failures and `LLMResponse.ok=false` both consume a retry.
- After `max_retries` exhausted: emit `extraction.failed` (per-chunk) → if every chunk failed, `pipeline.run_failed` (run terminal).
- M0.4 does NOT implement `extraction.dead_lettered` — that requires the Extractor to be invoked by an outbox subscriber, which is M0.5+ when ConflictProbe reads the same dispatcher path. P1's dead-letter is exclusively at the dispatcher level (already in M0.2).

```python
For every Python StrEnum member, assert that the C++ to_string equivalent
produces the same string. The C++ side is exercised through pybind11 (M0.4
Task 9 binds these enums); until then this test reads the C++ source via
a small regex helper to extract the string literals.

from __future__ import annotations

import re
from pathlib import Path

import pytest

from starling.schema.enums import (
    ConsolidationState, Modality, Perspective, Polarity,
    ReviewStatus, StatementProvenance,
)

ENUMS_CPP = (
    Path(__file__).resolve().parents[2] / "src" / "schema" / "statement_enums.cpp"
)


def _extract_pairs(enum_name: str) -> dict[str, str]:
    """Pull `case Foo::BAR: return "bar";` pairs out of the to_string switch."""
    text = ENUMS_CPP.read_text()
    pattern = re.compile(
        r"case\s+" + re.escape(enum_name) + r"::([A-Z_]+):\s*return\s+\"([^\"]+)\";"
    )

)
```

```cpp
#include "starling/extractor/extraction_span_key.hpp"

#include "starling/crypto/sha256.hpp"

#include <string>

namespace starling::extractor {

std::string compute_extraction_span_key(
        std::string_view engram_ref_id,
        std::int32_t     chunk_index,
        std::string_view predicate,
        std::string_view canonical_object_hash) {
    static constexpr char kSep = '\x1f';
    std::string canonical;
    canonical.reserve(engram_ref_id.size() + 16 + predicate.size()
                      + canonical_object_hash.size() + 4);
    canonical.append(engram_ref_id);
    canonical.push_back(kSep);
    canonical.append(std::to_string(chunk_index));
    canonical.push_back(kSep);
    canonical.append(predicate);
    canonical.push_back(kSep);
    canonical.append(canonical_object_hash);
    return starling::crypto::sha256_hex(canonical);
}

}  // namespace starling::extractor
```

```python
"""Python implementation of compute_extraction_span_key, for parity with the
C++ helper in src/extractor/extraction_span_key.cpp. The C++ side is the
canonical writer; this Python helper exists for prompt-input-hash construction
and for tests that round-trip canonical keys without crossing the C bridge."""
from __future__ import annotations

import hashlib

_SEP = b"\x1f"


def compute_extraction_span_key(
    engram_ref_id: str,
    chunk_index: int,
    predicate: str,
    canonical_object_hash: str,
) -> str:
    """Return the sha256 hex digest of the canonical span key.

    Mirror of starling::extractor::compute_extraction_span_key. The field
    order, separator, and chunk_index decimal encoding all match the C++
    implementation byte-for-byte.
    """
    canonical = (
        engram_ref_id.encode("utf-8")
        + _SEP
        + str(int(chunk_index)).encode("ascii")
        + _SEP
        + predicate.encode("utf-8")
        + _SEP
        + canonical_object_hash.encode("ascii")
    )
    return hashlib.sha256(canonical).hexdigest()
```

```cpp
#include "starling/extractor/xml_parser.hpp"

#include <algorithm>
#include <cctype>
#include <set>
#include <stdexcept>
#include <string>
#include <string_view>
#include <unordered_map>
#include <vector>

namespace starling::extractor {

namespace {

// Whitelisted tag set per M0.4 schema. Anything else → unknown_tag error.
const std::set<std::string> kKnownTags = {
    "extraction", "statement", "holder", "perspective", "subject",
    "predicate", "object", "modality", "polarity", "confidence",
    "observed_at", "perceived_by", "statement_ref",
};

// Whitelisted object kinds per §3.3 Value table. list / dict are explicitly
// rejected with kind="value_type_unsupported" so the LLM can't smuggle
// JSON-array objects through.
const std::set<std::string> kKnownObjectKinds = {
    "bool", "int", "float", "str", "datetime", "cognizer", "entity", "statement",
};

const std::set<std::string> kKnownPerspectives = {
    "first_person", "quoted", "hearsay", "inferred",
};

struct Element {
    std::string                                     name;
    std::unordered_map<std::string, std::string>    attrs;
    std::vector<Element>                            children;
    std::string                                     text;
};

// (1) Tokenize: scan for <tagname attr="..."...>, </tagname>, <tagname/> and
// text runs between them. Track byte offsets so ParseError.byte_offset is
// meaningful.
//
// (2) Build DOM: pop/push stack on open/close; reject unbalanced.
//
// (3) Walk DOM: validate root is <extraction>; for each <statement>,
// extract holder_ref, perspective, subject (kind+id), predicate text,
// object (kind+canonical_hash+inner text or <statement_ref id="..."/>),
// modality, polarity, confidence, observed_at, repeated <perceived_by ref/>.
// source_speaker on <statement> attribute is added to perceived_by if
// present and not already there. Validate against kKnownPerspectives,
// kKnownObjectKinds. Resolve short ids through ExistingRefMap.

// The implementer SHOULD split this into: tokenize() → DOM, then a
// separate validate_statement(Element) -> {ExtractedStatement, ParseError[]}.
// Both functions are pure; both are unit-testable; total file is ~350 LOC.

// Implementation skeleton (real bodies filled in by implementer):
std::vector<Element> tokenize(std::string_view xml,
                              std::vector<ParseError>& errors);

ExtractedStatement extract_statement(const Element& stmt_elem,
                                     const ExistingRefMap& refs,
                                     std::vector<ParseError>& errors);

}  // namespace

ParseResult parse_extractor_xml(
        std::string_view raw_xml,
        const ExistingRefMap& existing_ref_map) {
    ParseResult result;

    // Step 1: empty / whitespace-only check.
    bool any_non_ws = false;
    for (char c : raw_xml) {
        if (!std::isspace(static_cast<unsigned char>(c))) { any_non_ws = true; break; }
    }
    if (!any_non_ws) {
        result.errors.push_back({"empty_input", "no XML content", 0});
        return result;
    }

    // Step 2: tokenize + DOM build.
    std::vector<Element> roots = tokenize(raw_xml, result.errors);
    if (!result.errors.empty()) return result;
    if (roots.size() != 1 || roots.front().name != "extraction") {
        result.errors.push_back({"missing_root", "expected single <extraction> root", 0});
        return result;
    }

    // Step 3: per-statement extraction.
    for (const auto& child : roots.front().children) {
        if (child.name != "statement") {
            result.errors.push_back({"unknown_tag", "unexpected top-level: " + child.name, 0});
            continue;
        }
        std::vector<ParseError> stmt_errors;
        ExtractedStatement s = extract_statement(child, existing_ref_map, stmt_errors);
        if (!stmt_errors.empty()) {
            for (auto& e : stmt_errors) result.errors.push_back(std::move(e));
        } else {
            result.statements.push_back(std::move(s));
        }
    }

    return result;
}

}  // namespace starling::extractor
```

```cpp
    "bool", "int", "float", "str", "datetime", "cognizer", "entity", "statement",

bool is_weak_inference(const ExtractedStatement& s) {
    return s.holder_perspective == schema::Perspective::HEARSAY
        || s.holder_perspective == schema::Perspective::INFERRED
        || s.provenance == schema::StatementProvenance::TOM_INFERRED
        || s.confidence < 0.5;

ValidationOutcome validate_extracted_statement(const ExtractedStatement& s) {
    ValidationOutcome out;

    auto missing = [&](const char* field) {
        out.accepted = false;
        out.error_kind = "missing_required_field";
        out.detail = std::string("missing or empty: ") + field;
    };
    if (s.holder_id.empty())          { missing("holder_id");          return out; }
    if (s.holder_tenant_id.empty())   { missing("holder_tenant_id");   return out; }
    if (s.subject_kind.empty())       { missing("subject_kind");       return out; }
    if (s.subject_id.empty())         { missing("subject_id");         return out; }
    if (s.predicate.empty())          { missing("predicate");          return out; }
    if (s.object_kind.empty())        { missing("object_kind");        return out; }
    if (s.object_value.empty())       { missing("object_value");       return out; }
    if (s.canonical_object_hash.empty()) { missing("canonical_object_hash"); return out; }
    if (s.observed_at.empty())        { missing("observed_at");        return out; }
    if (s.source_hash.empty())        { missing("source_hash");        return out; }

    if (kAllowedObjectKinds.find(s.object_kind) == kAllowedObjectKinds.end()) {
        out.accepted = false;
        out.error_kind = "value_type_unsupported";
        out.detail = "object_kind not in {bool,int,float,str,datetime,cognizer,entity,statement}: " + s.object_kind;
        return out;
    }

    if (s.confidence < 0.0 || s.confidence > 1.0) {
        out.accepted = false;
        out.error_kind = "confidence_out_of_range";
        out.detail = "confidence must be in [0.0, 1.0]";
        return out;
    }
    if (s.confidence < 0.3) {
        out.accepted = false;
        out.error_kind = "below_minimum_confidence";
        out.detail = "confidence < 0.3 — extractor drops per §15.3.2";
        return out;
    }

    out.accepted = true;
    if (is_weak_inference(s)) {
        out.review_status_override = schema::ReviewStatus::INFERRED_UNREVIEWED;
    }
    return out;
}
```

```cpp
std::string random_id() {
    // Same scheme as PipelineLedger::random_id and OutboxWriter::random_event_id.
    // M0.4+1 will lift this into starling/util/uuid.hpp as real UUIDv7.
    static thread_local std::mt19937_64 rng{std::random_device{}()};
    const std::uint64_t a = rng(), b = rng();
    std::ostringstream oss;
    oss << std::hex << std::setfill('0')
        << std::setw(8) << static_cast<std::uint32_t>(a >> 32) << '-'
        << std::setw(4) << static_cast<std::uint16_t>(a >> 16) << '-'
        << std::setw(4) << static_cast<std::uint16_t>(a) << '-'
        << std::setw(4) << static_cast<std::uint16_t>(b >> 48) << '-'
        << std::setw(12) << (b & 0xFFFFFFFFFFFFULL);
    return oss.str();
}

// JSON escape for source_hash + UUIDs. P1 vocabulary is ASCII; no \uXXXX needed.
std::string json_string(std::string_view s) {
    std::string out;
    out.reserve(s.size() + 2);
    out.push_back('"');
    for (char c : s) {
        if      (c == '"')  out += "\\\"";
        else if (c == '\\') out += "\\\\";
        else if (c == '\n') out += "\\n";
        else if (c == '\r') out += "\\r";
        else if (c == '\t') out += "\\t";
        else if (static_cast<unsigned char>(c) < 0x20) { /* skip control */ }
        else                out.push_back(c);
    }
    out.push_back('"');
    return out;
}

std::string source_spans_json(const starling::extractor::ExtractedStatement& s,

// Look up an existing APPROVED statement matching the §15.3.2 chunk-duplicate
// criteria. Returns the existing stmt_id if found, otherwise empty string.
std::string find_existing_in_chunk(
        starling::persistence::Connection& conn,
        std::string_view tenant_id,
        std::string_view holder_id,
        std::string_view predicate,
        std::string_view canonical_object_hash,
        std::string_view evidence_engram_id) {
    sqlite3* db = conn.raw();
    sqlite3_stmt* raw = nullptr;
    if (sqlite3_prepare_v2(db,
        "SELECT id FROM statements "
        "WHERE tenant_id = ? AND holder_id = ? "
        "  AND predicate = ? AND canonical_object_hash = ? "
        "  AND evidence_json LIKE ? "
        "  AND review_status = 'approved' "
        "ORDER BY created_at ASC LIMIT 1",
        -1, &raw, nullptr) != SQLITE_OK) {
        throw make_sqlite_error(db, "find_existing_in_chunk: prepare");
    }
    StmtHandle h(raw);
    bind_sv(h.get(), 1, tenant_id);
    bind_sv(h.get(), 2, holder_id);
    bind_sv(h.get(), 3, predicate);
    bind_sv(h.get(), 4, canonical_object_hash);
    const std::string like_pat = std::string("%\"engram_ref\":\"") + std::string(evidence_engram_id) + "\"%";
    bind_sv(h.get(), 5, like_pat);
    const int rc = sqlite3_step(h.get());
    if (rc == SQLITE_DONE) return "";
    if (rc != SQLITE_ROW) throw make_sqlite_error(db, "find_existing_in_chunk: step");
    return reinterpret_cast<const char*>(sqlite3_column_text(h.get(), 0));
}

StatementWriteOutcome StatementWriter::write(
        const starling::extractor::ExtractedStatement& s,
        std::string_view evidence_engram_id,
        std::string_view extraction_span_key,
        std::optional<std::string> causation_parent_event_id) {

    const std::string stmt_id = random_id();

    // §15.3.2 chunk-level duplicate check.
    const std::string existing = find_existing_in_chunk(
        conn_, s.holder_tenant_id, s.holder_id, s.predicate,
        s.canonical_object_hash, evidence_engram_id);

    schema::ReviewStatus effective = s.review_status;
    if (!existing.empty()) {
        effective = schema::ReviewStatus::REVIEW_REQUESTED;
    }

    // For the evidence_json we need the engram's content_hash. Query for it
    // here rather than threading it through, so callers don't have to know.
    sqlite3* db = conn_.raw();
    std::string content_hash;
    {
        sqlite3_stmt* raw = nullptr;
        sqlite3_prepare_v2(db,
            "SELECT content_hash FROM engrams WHERE id = ? AND tenant_id = ?",
            -1, &raw, nullptr);
        StmtHandle h(raw);
        bind_sv(h.get(), 1, evidence_engram_id);
        bind_sv(h.get(), 2, s.holder_tenant_id);
        if (sqlite3_step(h.get()) == SQLITE_ROW) {
            content_hash = reinterpret_cast<const char*>(sqlite3_column_text(h.get(), 0));
        }
    }

    insert_statement_row(conn_, stmt_id, s, evidence_engram_id, content_hash, effective);

    // Build and append the bus_events row.
    const std::string canonical_key = std::string(extraction_span_key);
    const std::string causation_root = causation_parent_event_id.value_or("");
    const std::string window_bucket  = compute_window_bucket(
        "statement.written", std::chrono::system_clock::now());

    BusEvent ev;
    ev.tenant_id        = s.holder_tenant_id;
    ev.event_type       = "statement.written";
    ev.primary_id       = stmt_id;
    ev.aggregate_id     = stmt_id;
    if (causation_parent_event_id.has_value()) {
        ev.causation_chain.push_back(*causation_parent_event_id);
    }
    ev.idempotency_key  = compute_idempotency_key(
        "statement.written", stmt_id, canonical_key, causation_root, window_bucket);
    ev.payload_json     = statement_written_payload(
        stmt_id, s.holder_tenant_id, s.holder_id,
        schema::to_string(s.holder_perspective),
        s.predicate, s.canonical_object_hash,
        schema::to_string(effective), extraction_span_key,
        evidence_engram_id);

    OutboxWriter writer(conn_);
    writer.append(ev);

    if (!existing.empty()) {
        return StatementWriteChunkDuplicate{
            .stmt_id          = stmt_id,
            .original_stmt_id = existing,
            .event_id         = ev.event_id,
        };
    }
    return StatementWriteAccepted{
        .stmt_id         = stmt_id,
        .event_id        = ev.event_id,
        .outbox_sequence = ev.outbox_sequence,
    };
}
```

```cpp
struct TransactionGuard {
    sqlite3* db;
    bool committed = false;
    explicit TransactionGuard(sqlite3* d) : db(d) {
        sqlite3_exec(db, "BEGIN IMMEDIATE", nullptr, nullptr, nullptr);
    }
    void commit() {
        sqlite3_exec(db, "COMMIT", nullptr, nullptr, nullptr);
        committed = true;
    }

        if (!committed) sqlite3_exec(db, "ROLLBACK", nullptr, nullptr, nullptr);
    }

ExtractionRunResult Extractor::run(
        std::string_view                  engram_ref_id,
        const std::vector<std::uint8_t>&  payload_bytes,
        std::string_view                  holder_id,
        std::string_view                  holder_tenant_id,
        const ExistingRefMap&             existing_ref_map) {

    ExtractionRunResult result;

    // The orchestrator uses a single transaction per run. The retry loop
    // re-prepares the LLM call but stays inside the same transaction; on
    // FAILED outcome we still COMMIT (so attempt rows + events persist),
    // just with terminal status='failed'. On exception we ROLLBACK
    // (TransactionGuard does it).
    sqlite3* const db = conn_.raw();
    TransactionGuard tx(db);

    PipelineLedger ledger(conn_);
    const std::string run_id = ledger.start_run(holder_tenant_id, engram_ref_id, "{}");
    const std::string run_started_event_id = emit_pipeline_event(
        conn_, "pipeline.run_started", holder_tenant_id, run_id, std::nullopt);
    result.run_id = run_id;

    const std::int32_t chunk_index = 0;  // M0.4: 1 chunk per Engram
    const std::string chunk_span_key = compute_extraction_span_key(
        engram_ref_id, chunk_index, "__chunk__", "__chunk__");

    bool any_accepted = false;
    bool all_failed   = true;

    const std::string prompt_body = build_prompt_body(
        holder_id, payload_bytes, existing_ref_map);
    const std::string prompt_input_hash = compute_prompt_input_hash(prompt_body);

    for (int attempt = 1; attempt <= kMaxRetries; ++attempt) {
        const LLMResponse resp = adapter_.extract(prompt_body, prompt_input_hash);
        if (!resp.ok) {
            ledger.record_attempt(run_id, chunk_span_key, attempt,
                                  ExtractionStatus::Failed,
                                  /*raw_output=*/{},
                                  resp.error);
            emit_extraction_event(conn_, "extraction.failed",
                                  holder_tenant_id, run_id, chunk_span_key,
                                  run_started_event_id);
            if (attempt < kMaxRetries) {
                emit_extraction_event(conn_, "extraction.retry_scheduled",
                                      holder_tenant_id, run_id, chunk_span_key,
                                      run_started_event_id);
            }
            continue;
        }

        ParseResult parsed = parse_extractor_xml(resp.raw_xml, existing_ref_map);
        if (!parsed.errors.empty()) {
            ledger.record_attempt(run_id, chunk_span_key, attempt,
                                  ExtractionStatus::Failed,
                                  resp.raw_xml,
                                  parsed.errors.front().kind);
            emit_extraction_event(conn_, "extraction.failed",
                                  holder_tenant_id, run_id, chunk_span_key,
                                  run_started_event_id);
            if (attempt < kMaxRetries) {
                emit_extraction_event(conn_, "extraction.retry_scheduled",
                                      holder_tenant_id, run_id, chunk_span_key,
                                      run_started_event_id);
            }
            continue;
        }

        // Parse succeeded. Validate and write each statement.
        StatementWriter writer(conn_);
        bool any_rejected = false;
        bool wrote_anything_this_attempt = false;
        for (auto& stmt : parsed.statements) {
            stmt.holder_id        = std::string(holder_id);
            stmt.holder_tenant_id = std::string(holder_tenant_id);
            stmt.chunk_index      = chunk_index;
            if (stmt.source_hash.empty()) {
                stmt.source_hash = "chunk-" + std::to_string(chunk_index);
            }
            const ValidationOutcome v = validate_extracted_statement(stmt);
            if (!v.ok()) {
                any_rejected = true;
                continue;
            }
            if (v.review_status_override.has_value()) {
                stmt.review_status = *v.review_status_override;
            }

            const std::string span_key = compute_extraction_span_key(
                engram_ref_id, chunk_index, stmt.predicate, stmt.canonical_object_hash);
            if (extraction_span_key_already_succeeded(conn_, span_key)) {
                ledger.record_attempt(run_id, span_key, attempt,
                                      ExtractionStatus::Success,
                                      /*raw_output=*/{},
                                      /*error=*/"noop:extraction_span_key_hit");
                emit_extraction_event(conn_, "extraction.noop",
                                      holder_tenant_id, run_id, span_key,
                                      run_started_event_id);
                continue;
            }

            const auto outcome = writer.write(stmt, engram_ref_id, span_key, run_started_event_id);
            std::string accepted_id;
            if (auto* ok = std::get_if<StatementWriteAccepted>(&outcome)) {
                accepted_id = ok->stmt_id;
            } else {
                accepted_id = std::get<StatementWriteChunkDuplicate>(outcome).stmt_id;
            }
            result.accepted_statement_ids.push_back(accepted_id);
            wrote_anything_this_attempt = true;
        }

        const auto attempt_status =
            wrote_anything_this_attempt
                ? (any_rejected ? ExtractionStatus::PartialSuccess
                                : ExtractionStatus::Success)
                : (any_rejected ? ExtractionStatus::Failed
                                : ExtractionStatus::Success);  // empty = noop, treat success
        ledger.record_attempt(run_id, chunk_span_key, attempt,
                              attempt_status,
                              resp.raw_xml,
                              /*error=*/{});
        if (wrote_anything_this_attempt) any_accepted = true;
        if (wrote_anything_this_attempt || !any_rejected) {
            all_failed = false;
            break;  // success path; no retry
        }
        all_failed = false;  // had a parse, just nothing accepted; not a transient error → no retry
        break;
    }

    if (any_accepted) {
        result.status = ExtractionRunResult::Status::SUCCESS;
        ledger.finish_run(run_id, PipelineStatus::Finished);
        emit_pipeline_event(conn_, "pipeline.run_completed",
                            holder_tenant_id, run_id, run_started_event_id);
    } else if (all_failed) {
        result.status = ExtractionRunResult::Status::FAILED;
        ledger.finish_run(run_id, PipelineStatus::Failed);
        emit_pipeline_event(conn_, "pipeline.run_failed",
                            holder_tenant_id, run_id, run_started_event_id);
    } else {
        // No accepted statements but not transient-failure either (validator
        // rejected every candidate). Treat as a finished run with no output.
        result.status = ExtractionRunResult::Status::SUCCESS;
        ledger.finish_run(run_id, PipelineStatus::Finished);
        emit_pipeline_event(conn_, "pipeline.run_completed",
                            holder_tenant_id, run_id, run_started_event_id);
    }

    tx.commit();
    return result;
}

}  // namespace starling::extractor
```
