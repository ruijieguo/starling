# M0.1 Schema 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 落地 Starling Memory 数据本体（Schema）：16 个 enum、~13 个数据类（含 ~40 字段的 Statement）、Container 三态分区 key、`canonicalize_object()` v1，配套 Python binding 与 JSON-Schema 导出，作为后续所有子系统（M0.2 持久化、M0.3 EngramStore、M0.4 Extractor、M0.5 ConflictProbe、M0.6 Retrieval）的契约层。

**Architecture:** Schema 以纯 Python `@dataclass(frozen=True, slots=True)` 为单一事实源，C++ 仅暴露 `canonicalize_object()` 与 enum 镜像（性能关键路径）；validators 是独立函数（非方法）便于组合；JSON-Schema 通过反射自动导出，供 M0.2 SQLite DDL 与 M0.4 LLM Extractor 共享；本里程碑只定义类型 + validator + canonicalize，不写持久化、不写 `Bus.write`、不写 `rebuild_container` 运行时。

**Tech Stack:**
- Python 3.11+（`@dataclass(frozen=True, slots=True, kw_only=True)`、`StrEnum`、`typing.Annotated`）
- C++20（仅 `canonicalize_object` 与 enum 镜像，pybind11 暴露）
- pytest 8.x + hypothesis 6.x（property-based testing for canonicalize）
- jsonschema 4.x（导出验证）
- 不引入 ORM / 不写 SQL（M0.2 才接入 SQLite）

**M0.1 不涉及**：SQLite DDL（M0.2）、outbox / dispatcher（M0.2）、`Bus.append_evidence`（M0.3）、LLM 调用（M0.4）、ConflictProbe 真实冲突算子（M0.5，但提供 `canonicalize_object` 这一前置依赖）、`rebuild_container` 运行时（P2）、Replay/Reconsolidation/Prospective（P2）、Retrieval Planner（P3）。

```python
"""Typed reference wrappers (M0.1).

Each Ref class wraps a UUID and is `eq=True`/`frozen=True`/`slots=True`. The
six classes do NOT share a base — cross-type equality is intentionally false
so that mixing CognizerRef with EntityRef in a dict cannot collide.
"""

from __future__ import annotations
import uuid
from dataclasses import dataclass
from typing import ClassVar


def _ref_class(name: str):
    """Define a frozen-slots dataclass that wraps a UUID under a unique name."""
    @dataclass(frozen=True, slots=True)
    class _Ref:
        id: uuid.UUID

        def __str__(self) -> str:
            return str(self.id)

        @classmethod
        def from_str(cls, raw: str):
            return cls(uuid.UUID(raw))

    _Ref.__name__ = name
    _Ref.__qualname__ = name
    return _Ref


CognizerRef = _ref_class("CognizerRef")
EntityRef = _ref_class("EntityRef")
StatementRef = _ref_class("StatementRef")
EngramRef = _ref_class("EngramRef")
PersonaRef = _ref_class("PersonaRef")
KnowledgeFrontierRef = _ref_class("KnowledgeFrontierRef")
```

## Task 3: canonicalize_object v1（Python 参考实现）

**Files:**

§3.3 Value standardization 表是契约源。`canonicalize_object` v1 接受 `bool | int | float | str | datetime | CognizerRef | EntityRef | StatementRef`，返回 `(canonical_string, hash_hex)`：

| 输入类型 | 标准化 |
|---|---|
| `bool` | `"true"` / `"false"` |
| `int` | 十进制；负数前置 `-`；不带分组符；不带 `+` |
| `float` | 定点 6 位小数（`f"{x:.6f}"`）；`-0.0 → "0.000000"`；`NaN`/`Inf` 拒收（`ValueError`） |
| `str` | NFC → trim → 连续 whitespace 折叠为单空格 → lowercase（保留 CJK 不变换大小写，但 NFC 仍生效） |
| `datetime` | tz-aware 必填；转 UTC；`strftime("%Y-%m-%dT%H:%M:%SZ")` |
| `Ref` | `f"{ref-class-name}:{uuid_hex}"`（如 `"CognizerRef:550e8400e29b41d4a716446655440000"`） |
| `list` / `dict` / `None` / 其他 | `ValueError("schema_invalid")` |

hash 算法：`sha256(canonical_string.encode("utf-8")).hexdigest()`，返回 `(canonical, hash_hex)`，调用方按需拼版本：`Statement.canonical_object_hash = hash_hex`，`canonical_object_hash_version = "v1"`。

**str 大小写说明**：§3.3 原文「有大小写语义文本执行 lowercase」。v1 简化为对所有 `str` 输入统一 `.lower()`（CJK 字符 `.lower()` 是 no-op，与原文一致）。M0.5 ConflictProbe 若需保留专有名词大小写，应通过 Statement 子类型或附加字段处理，不改 v1 规则。

```python
def _hash_of(canon: str) -> str:
```

```python
"""canonicalize_object v1 — produces a (canonical_string, sha256_hex) pair.

Used by Statement.canonical_object_hash (set at write time, M0.5
ConflictProbe indexes it). Version field on Statement
(canonical_object_hash_version) lets us evolve the rules; this module
implements v1 only.
"""

from __future__ import annotations
import hashlib
import math
import re
import unicodedata
import uuid
from datetime import datetime, timezone

from starling.schema.refs import (
    CognizerRef, EntityRef, StatementRef, EngramRef,
    PersonaRef, KnowledgeFrontierRef,
)

_REF_CLASSES = (
    CognizerRef, EntityRef, StatementRef,
    EngramRef, PersonaRef, KnowledgeFrontierRef,
)

_WHITESPACE_RUN = re.compile(r"\s+")

CanonicalizableValue = (
    bool | int | float | str | datetime
    | CognizerRef | EntityRef | StatementRef
    | EngramRef | PersonaRef | KnowledgeFrontierRef
)


def canonicalize_object(value) -> tuple[str, str]:
    canonical = _to_canonical(value)
    digest = hashlib.sha256(canonical.encode("utf-8")).hexdigest()
    return canonical, digest


def _to_canonical(value) -> str:
    # IMPORTANT: bool must be checked before int because bool is a subclass of int.
    if isinstance(value, bool):
        return "true" if value else "false"
    if isinstance(value, int):
        return str(value)
    if isinstance(value, float):
        if math.isnan(value) or math.isinf(value):
            raise ValueError("schema_invalid: NaN/Inf not canonicalizable")
        # Collapse -0.0 to 0.0 before formatting.
        if value == 0.0:
            value = 0.0
        return f"{value:.6f}"
    if isinstance(value, str):
        nfc = unicodedata.normalize("NFC", value)
        trimmed = nfc.strip()
        folded = _WHITESPACE_RUN.sub(" ", trimmed)
        return folded.lower()
    if isinstance(value, datetime):
        if value.tzinfo is None:
            raise ValueError("schema_invalid: naive datetime not canonicalizable")
        utc = value.astimezone(timezone.utc)
        return utc.strftime("%Y-%m-%dT%H:%M:%SZ")
    if isinstance(value, _REF_CLASSES):
        return f"{type(value).__name__}:{value.id.hex}"
    raise ValueError(f"schema_invalid: unsupported type {type(value).__name__}")
```

```python
@dataclass(frozen=True, slots=True, kw_only=True)
class AffectVector:
    valence: float        # -1..+1
    arousal: float        #  0..1
    dominance: float      # -1..+1
    novelty: float        #  0..1
    stakes: float         #  0..1

    def salience(self, surprise_decay: float = 1.0) -> float:
        return (
            (0.4 + 0.6 * abs(self.valence))
            * (0.4 + 0.6 * self.arousal)

        )
```

```python
class SchemaInvalid(ValueError):
    """Raised when a Statement / Container fails a cross-field invariant."""

# 1. evidence ∨ derived_from 至少其一非空 + 与 provenance 兼容（§3.3）
def validate_evidence_or_derivation(stmt: Statement) -> None: ...

# 2. subject 不能是 StatementRef（§3.2 / §3.3 hash 稳定性）
def validate_subject_not_statement(stmt: Statement) -> None: ...

# 3. derived_depth = max(parent.derived_depth)+1 当 derived_from 非空（§3.3）
#    M0.1 不能跨 Statement 校验（没有持久化），所以此函数接受 (stmt, parent_depths: dict[StatementRef, int])
def validate_derived_depth(stmt: Statement, parent_depths: dict) -> None: ...

# 4. tenant_id 派生 + 不可变（§3.2）：写入路径单独校验
def validate_tenant_derived(stmt: Statement, holder: Cognizer) -> None: ...

# 5. holder_perspective + provenance 兼容（§3.3 默认入态表）
def validate_perspective_provenance(stmt: Statement) -> None: ...

# 6. canonical_object_hash 一致性（§3.3）：与 canonicalize_object(object) 算出的 hash 必须相等
def validate_canonical_object_hash(stmt: Statement) -> None: ...

# 7. evidence 的 EvidenceStatus 兼容性：ACTIVE -> erased_at must be None；ERASED -> erased_at not None
def validate_evidence_status(stmt: Statement) -> None: ...

# 8. Container dimension key 合法性（§3.7）
def validate_container_dimension(container_kind: ContainerKind, dimension: str) -> None: ...
```

每个 dataclass 反射出 JSON-Schema draft-2020-12，提供两种入口：

```python
def schema_for(cls: type) -> dict: ...
def all_schemas() -> dict[str, dict]: ...        # 所有 13 个数据类的 schema 包
```

实现细节：手写一个最小反射器（不引入 pydantic / msgspec），覆盖类型：
- 基本类型（bool/int/float/str → JSON-Schema type）
- `datetime` → `{"type": "string", "format": "date-time"}`
- `uuid.UUID` → `{"type": "string", "format": "uuid"}`
- `Literal[...]` → `{"enum": [...]}`
- `tuple[T, ...]` → `{"type": "array", "items": schema_for(T)}`
- `Ref` 类 → `{"type": "string", "format": "uuid"}`
- StrEnum 类 → **inlined** `{"type": "string", "enum": [member.value, ...]}`（不用 `$ref`，测试断言依赖 `properties[k]["enum"]` 直接可读）
- `Optional[T]` / `T | None` → `{"oneOf": [schema_for(T), {"type": "null"}]}`
- `Union[A, B, ...]` → `{"oneOf": [schema_for(A), schema_for(B), ...]}`

**目的**：M0.2 SQLite DDL 与 M0.4 LLM Extractor 的 strict prompt 都从这里读契约（不重复维护）；M0.7 acceptance 用 `jsonschema.validate` 做端到端 sanity check。

```cpp
// include/starling/schema/canonicalize.hpp
#pragma once
#include <chrono>
#include <cstdint>
#include <span>
#include <string>
#include <variant>

namespace starling::schema {

struct CanonicalRefInput {
    std::string class_name;       // "CognizerRef" 等
    std::array<std::uint8_t, 16> uuid_bytes;
};

using CanonicalInput = std::variant<
    bool, std::int64_t, double, std::string,
    std::chrono::sys_seconds,     // tz-aware UTC seconds
    CanonicalRefInput
>;

struct CanonicalResult {
    std::string canonical;
    std::string sha256_hex;
};

CanonicalResult canonicalize_object(const CanonicalInput& v);

}  // namespace starling::schema
```
