# Starling Memory 实现路线图（Roadmap Index）

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement each milestone plan task-by-task.
> 本文件是**索引**，不直接执行。每个里程碑配独立 plan 文件，按下表入口找。

**Goal:** 按 P1（最小闭环）→ P2（社会心智 + 类脑动力学 + 前瞻）→ P3（完整 Retrieval Planner + 二阶 ToM + 多底座产品化）三阶 32 周实现 Starling Memory。

**Architecture:** C++20 内核 + pybind11 绑定（NAPI / cffi 按需，P3）；三档 Substrate Profile——P1/P2 已落地 local-store = raw SQLite（WAL 单写者）+ SqliteBlobVectorIndex（暴力 BLOB 向量，经 VectorIndex seam），seekdb/LadybugDB 与 dist-store（Postgres+pgvector+AGE）、cloud-store 三形态顺延 P3.b；Statement Bus + outbox 单一写入入口；六态 ConsolidationState 状态机驱动 Hippocampus / Neocortex 双库分区。

**Tech Stack:**
- 内核：C++20（CMake + Ninja）
- 绑定：pybind11（已落地）；NAPI（JS）/ cffi（Rust/C）按需（P3）
- 存储 P1/P2（已落地）：raw SQLite + SqliteBlobVectorIndex（seekdb/LadybugDB 目标形态顺延 P3.b）
- 存储 P3.b：seekdb / LadybugDB（local）、Postgres 18 + pgvector + Apache AGE（dist）、单云原生 / 混合托管 / 跨云 SaaS（cloud）
- 测试：GoogleTest（C++）+ pytest（Python）+ clang-tidy + ASan/UBSan
- LLM：通过 Adapter 抽象（OpenAI / Anthropic / 本地兼容）
- 评测：LongMemEval / ToMBench / FANToM / SoMi-ToM

## 为什么本文件不是单一巨型 plan

writing-plans skill 的硬约束："Each task should produce self-contained changes". 本工程 12 子系统 × 32 周，把所有里程碑塞一份文件等价于在不知道契约具体形态时凭空写步骤；后续里程碑的 schema、API、测试都依赖前一里程碑落地后的真实代码。

因此本文件作 **roadmap 索引 + 拆分原则 + 进度追踪**，每个里程碑落地前才生成对应 plan。**现状（2026-06-10）**：本目录 25 份 plan 已全部写就并完成实施（P1 M0.0–M0.7、P2 P2.a–P2.c 含 M0.8/M0.9、P2 收尾 P2.d–P2.m、build-dependency-config），P3 plan 待写。本文件继续作为索引与进度账本维护。
