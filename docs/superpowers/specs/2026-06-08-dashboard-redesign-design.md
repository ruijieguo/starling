## 0. 背景与问题（现状诊断）

P2.g/P2.h 落地了一个**能跑的 dashboard 功能原型**：FastAPI engine-API 后端 + SvelteKit(Svelte 5)/Tailwind v4 前端，单进程单端口 serve，`#token` 片段登录，`~/.starling/starling.json`(0600) 统一配置，12 个面板平铺。但用户评价"太糙、功能简陋、LLM/embedding 不支持配置"。逐项核查（读真实代码）确认：

- **栈现代、皮全裸**：`dashboard/web/src/app.css` 仅 4 行，无任何设计 token；整套纯 zinc 灰阶，颜色只在"绿=已配/红=错"出现；无字体系统、无层级、无性格。是线框图不是产品。
- **信息架构平摊**：`+layout.svelte` 侧栏 12 项平铺，无分组、无当前页高亮；token 输入框塞在侧栏底部，无标签、无粘贴提示。
- **配置体验最弱（用户点名）**：`/settings` 技术上能配 LLM+embedder，但全是裸文本框——无"测试连接"（填错要等 `remember` 报错才知）、无 provider 预设/模型下拉、后端**只支持 OpenAI 兼容一种**、api_key 无显示切换、保存只闪一下无 toast/校验、改 embedder 重嵌**无确认弹窗**；后端更隐蔽：无 key 时 embedder **静默降级成 8 维 stub**，UI 不提示；重嵌是同步、破坏性、无进度。
- **数据展示一律最朴素**：`DataTable` 把嵌套对象 `JSON.stringify` 塞单元格，无排序/筛选/分页；`Graph` 写死圆形布局 >20 节点即糊；Eval/Replay 直接 `<pre>` 倒原始 JSON/markdown；几乎所有页面只处理 error、无 loading 态。
- **数据层薄**：`api.ts` 无重试/超时/401 处理；`ws.ts` 无重连/心跳。

定性：**有能力、无产品**——没有视觉语言、没有反馈闭环、配置不可信、表格不可用。本设计把它升级成有品味、可信赖、两类用户都顺手的工作台。

## 1. 产品定位与用户

**定位**：一个有品味、可信赖的**记忆体工作台**，同时服务两类用户。

| 用户 | 是谁 | 核心 job-to-be-done |
|---|---|---|
| 终端用户 | 用这套记忆系统的人 | 看 agent 记住了什么 / 此刻在想什么 / 欠谁承诺；直接 remember、recall、render working set |
| 开发者 | 把 Starling 接进自己系统的工程师 | 可靠地配 LLM/embedder（测连通、选模型、换 provider）；诊断抽取/冲突/队列；端到端看清数据流 |

**主线锚点**：两者并重，**配置体验做透**作为第一块硬骨头，并借此立起**统一设计语言**贯穿全局。

---

## 2. 决策摘要（已锁定）

1. **定位**：两者并重，配置体验第一块硬骨头 + 统一设计语言贯穿。
2. **配置深度**：完整版——可插拔 provider 工厂 + **Anthropic 原生适配器**（新 C++/绑定）+ OpenAI 兼容预设全覆盖 + 测试连接 + 校验 + 反馈。
3. **范围**：全 12 面板深做 + IA 重组 + 设计系统。
4. **视觉**：**克制现代**（Linear/Vercel 系）——中性 slate + 单一品牌色 indigo，明暗等价，组件精致，**密度按面板可调**。
5. **分期**：**地基优先**——P2.k 设计系统/壳/IA（纯前端，可在 main 做，零 C++）→ P2.l 配置深做 + 多适配器后端（C++，worktree+重建）→ P2.m 12 面板按 IA 分组深做。
6. **命名**：接项目里程碑字母 **P2.k / P2.l / P2.m**（P2.g/P2.h 即前两期 dashboard）。

## 3. 信息架构（12 平铺 → 4 组）

| 组 | 面板（现有路由） | 主服务 |
|---|---|---|
| **观测 Observe** | Overview（首页 `/`）· Statements `/statements` · Cognizers `/cognizers` · Commitments `/commitments` | 终端用户为主，开发者只读检视 |
| **交互 Interact** | Interact `/interact` · Working Set `/working-set` · Reminders `/reminders` | 终端用户 |
| **诊断 Diagnose** | Queues `/queues` · Conflicts `/conflicts` · Replay `/replay` · Eval `/eval` | 开发者 |
| **设置 Settings** | `/settings`（标签页：Provider(LLM) / Embedder / Access / About） | 开发者，首跑必经 |

合计 4 + 3 + 4 + 1 = 12，无遗漏。Overview 既是"观测"首项，也是默认落地路由 `/`。

**新壳**：
- 分组侧栏（组标题 + 缩进项）+ **当前页高亮**（按 `$page.url.pathname` 匹配）。
- 顶部状态区：**连接 ●** / **LLM ●** / **Embedder ●** 三盏健康灯 + 最近 tick 时间。健康灯颜色用语义色（绿/琥珀/红）但同时带文字/图标（颜色不作唯一信号）。
- Token 从侧栏底部迁入 **设置 → Access**（带"复制"、"从 #token URL 粘贴"说明、设置态指示）。
- 响应式：窄屏侧栏可收起为抽屉；桌面为主。

## 4. 设计系统（克制现代 · slate + indigo）

### 4.1 Tokens
- **色**：中性 slate 阶（背景/文本/边框）+ 单一品牌色 **indigo #4F46E5**（主操作/选中/链接）+ 语义色 success(green)/warn(amber)/danger(red)/info(blue)，语义色克制使用。**明暗等价**：用 CSS 变量定义 token，明/暗两套值；Tailwind `dark:` 仅作兜底。
- **字体**：`Inter`（变量字体，UI 全栈）+ `ui-monospace`/`JetBrains Mono`（ID/hash/JSON/code）。字号阶（display/title/body/caption）。
- **间距/圆角/投影**：4px 基；radius md/lg/xl；elevation token（柔和阴影，暗色下用边框+微光）。
- **密度模式**：面板根上 `data-density="comfortable|compact"`；同一套 token，`compact` 收紧行高/内边距（诊断面板/大表格用），`comfortable` 默认（叙事面板用）。

### 4.2 组件库（`src/lib/components/ui/`）
基础件（手写）：Button · IconButton · Input · Textarea · **Field**(label+hint+error 包裹) · Switch/Checkbox · Badge · **StatusDot**（带 label 的语义灯）· Card · CopyButton · **EmptyState** · **Skeleton**。

交互原语（**bits-ui**：Svelte 5 headless 无障碍原语，用我们的 token 上皮，shadcn 风）：**Dialog/Modal** · **Select/Combobox** · Tooltip · Tabs · DropdownMenu · Toast(`Toaster`)。

数据件（升级）：
- **StatCard**：label + 大数 + 可选 trend（↑/↓ + 对比）+ 可选 tooltip 释义。
- **DataTable**：列定义驱动；排序 / 客户端筛选 / 分页；**单元格格式化器**（对象→徽章/截断+展开抽屉，而非 `JSON.stringify`）；空态 / 载入 skeleton；可访问（`scope`、`aria-sort`）。
- **CodeBlock**：格式化展示 JSON / markdown（替代裸 `<pre>`），可折叠、可复制、等宽。
- **Graph**：改进社会图（更好的布局：tidy/力导向二选一由 plan 定；hover tooltip；点击→节点详情；标签防重叠；可缩放/平移）。

**架构决策（组件原语）**：交互/无障碍重的件用 **bits-ui** 而非全手写——换来 *无障碍 + 控制 + 速度*，不背重型不可控 UI 套件（如 DaisyUI/Skeleton 主题）。简单件手写保持轻与一致。备选：全手写（更可控但 a11y 负担重、慢）——不采。

### 4.3 横切状态（每个异步视图）
载入(skeleton) / 空态(说明性，非空白) / 错误态(可重试，含上下文而非裸 `${status}`) / 成功反馈(toast)。统一封装一个数据加载原语（如 `createQuery(fetchFn)` store 暴露 `{data, error, loading, refetch}`），各页面复用，消除当前每页 ~50 行样板。

## 5. 配置体验（硬骨头）+ 多适配器后端

### 5.1 配置模型 v2（引入 `provider`）
- **Chat**：`{ provider, model, base_url, api_key }`，`provider ∈ {openai, azure, ollama, vllm, lmstudio, groq, deepseek, openrouter, anthropic, custom}`。
- **Embedder**：`{ provider, model, base_url, api_key, dim }`，`provider ∈ {openai, azure, ollama, voyage, custom}`。**Anthropic 无 embedding API**，故 embedder provider 列表**不含 anthropic**，UI 明示并建议 Voyage 或 OpenAI 兼容。
- **预设表**（前端常量 + 后端校验共识）：每个 provider 定义默认 `base_url`、常用模型下拉候选、`needs_key`(bool)。选 provider 自动填 `base_url` + 模型下拉 + 标注是否需 key。`custom` 全手填。
- **向后兼容**：旧配置（无 `provider` 字段，仅 model/base_url/api_key）读入时推断为 `openai`（若 base_url 空或指向 openai）或 `custom`，不破坏 `~/.starling/starling.json` 既有文件。

### 5.2 后端适配器
- **OpenAI 兼容的全部 provider**（openai/azure/ollama/vllm/lmstudio/groq/deepseek/openrouter/custom）→ 复用现有 `_core.OpenAIAdapter`，仅 `base_url`/`model` 不同。**零新 C++**。
- **Anthropic** → **新 `_core.AnthropicAdapter`**（C++，实现与 `OpenAIAdapter` 相同的 LLM 适配器接口）：`POST {base_url}/v1/messages`，头 `x-api-key` + `anthropic-version`，体 `{model, max_tokens, system?, messages:[{role:"user", content:<prompt>}]}`，解析 `content[].text` 拼接。**api_key 仅来自环境**（`ANTHROPIC_API_KEY`，经 env-swap 注入，镜像现有 OpenAI 做法），绝不入参/日志/绑定/提交。pybind 绑定暴露为 `_core.AnthropicAdapter`（接口与 `_core.OpenAIAdapter` 对齐）。
  - *P2.l plan 须用代码锚点确认*：现有 C++ LLM 适配器接口的确切名/虚函数（`OpenAIAdapter`/`FakeLLMAdapter` 的共同基类与 `complete`/`run` 签名）、`OpenAIAdapter` 的 libcurl 调用与 JSON 组装位置、`bindings/python/module.cpp` 适配器绑定写法、`OpenAIAdapterConfig`/env 读取模式——`AnthropicAdapter` 照其模式实现，复用 libcurl + nlohmann/json。
- **Embedder**：OpenAI 兼容 → `_core.OpenAIEmbeddingAdapter`；Voyage（可选）→ 评估其 API 是否 OpenAI 兼容（多数兼容 `/embeddings` 形态），兼容则走现有适配器仅换 base_url/model，不兼容则标为"未来"，本设计不强制实现。无 key → 仍是 `StubEmbeddingAdapter`，但**显式可见**（见 5.4）。
- **Provider 工厂放 Python `DashboardEngine`**：`_build_chat_adapter(cfg)` 按 `cfg.provider` 分发（`anthropic` → `AnthropicAdapter`，余 → `OpenAIAdapter`）；`_build_embedder(cfg)` 类似。C++ 改动收敛为"只加 `AnthropicAdapter` 一个类 + 绑定"，最小化碰 core。

### 5.3 测试连接（让配置可信的关键）
新端点 **`POST /api/config/test`**：收候选 provider 配置（含可能的明文 key，仅本机 localhost），用它建**临时适配器**、真打一次最小调用——
- chat：极短 1-token 补全（或最廉价的合法请求）；
- embedder：`embed("ping")`，回 dim。
返回 `{ ok, latency_ms, detail, model_echo?, dim? }`。**不落盘、不持久化、不回传 key、不写日志**（确保该端点请求体的 key 不进任何 log）。前端在 Provider/Embedder 标签页提供"测试连接"按钮 + 内联结果（绿勾/红叉 + 延迟 + 失败原因）。**保存前可先测**。

### 5.4 校验、守护、降级可见
- **字段级校验**：按 provider 必填（如 openai/anthropic/groq 等需 key+model；ollama/lmstudio 需 base_url、key 可空）。提交前校验，错误定位到字段（红框 + 文案），不再裸 `400`。
- **embedder dim 守护**：改 embedder 时读当前已存向量维度（DB），与新 `dim` 比对；不一致才弹**重嵌确认弹窗**（明示将重嵌 N 条记忆）。重嵌从"同步阻塞静默"改为**显式 + 带进度**（复用 tick/queue 推进，UI 显示进度，不阻塞请求线程）。
- **降级可见**：无 key / embedder 为 8 维 stub 时，顶部 **Embedder 灯转琥珀** + 设置页明示"未启用(stub)，召回不可用"，不再静默。
- **保存反馈**：toast（持久数秒）替代一闪即逝的文字。

### 5.5 安全（不放松既有红线）
- key 仅在 env-swap + `~/.starling/starling.json`(0600) + 进程内存；`GET /api/config` 只回 `key_set`(bool)，绝不回 token/全 key。
- `POST /api/config/test` 仅本机收明文 key（与 `POST /api/config` 同模型），且**确保不记录请求体**。
- 可选：Access 页加"轮换 token"（重新生成 + 重打印 `#token` URL）。
- 不改 `ci_static_scan.py` 守护的红线；新增端点纳入 dashboard auth 测试。

## 6. 各面板深做（P2.m，按 IA 分组）

### 观测 Observe
- **Overview**：状态 hero（连接 / LLM / Embedder 健康 + 最近 tick）+ 关键计数升级 StatCard（带 trend）+ 承诺状态条（按状态分布的可视条）+ **实时活动流**（经 `/ws` tick/statement_added 增量）+ 快捷入口。空/载入 skeleton。
- **Statements**：真 DataTable（排序/分页）+ **faceted 筛选**（predicate / perspective / modality / polarity）+ 行 → **详情抽屉**（holder、perceived_by、scope_parties、affect、出处 engram；可读格式化而非裸 JSON）。
- **Cognizers**：改进社会图（更好布局 + hover tooltip + 点击节点详情）+ presence/relations + 表格兜底。
- **Commitments**：状态机**泳道**（按状态分列）+ 筛选 + 截止排序 + `fired` ⚠ 醒目 + 行 → 详情。

### 交互 Interact
- **Interact**：真"控制台"。remember（textarea + holder/interlocutor，结果把抽取出的 statements 渲染为可钻入 chips）；recall（query + **语义/补全模式切换** + 结果为带分数的可读卡片）；载入态、复制 affordance。
- **Working Set**：interlocutor + goal 输入 + 渲染上下文块**分区高亮**（persona / common_ground / relevant_memories / pending_commitments / affect）+ **token 预算计** + `⚠ DUE` 提醒醒目 + **一键复制为 prompt**。
- **Reminders（承诺提醒）**：按截止排序的待办（active/created）+ ⚠ 状态 + 清晰空态（"无待办承诺"）。

### 诊断 Diagnose
- **Queues**：embedding 积压 / dispatch / 向量状态，**健康框定**（绿/琥珀/红）+ 趋势 + 手动 "tick" 按钮 + 积压告警。
- **Conflicts**：按 kind 分组 + 权重**可视条** + 筛选 + src/dst 可读 + **链接到相关语句**。
- **Replay**：scheduler 状态渲染为**结构化卡片**（替代 `JSON.stringify` 裸倒）+ ledger / windows 表。
- **Eval**：**解析后的 markdown** 报告（替代 `<pre>` 倒灌）+ 可折叠 + 最新优先 + 运行元信息。

## 7. 横切（数据层 + 无障碍）

- **数据层加固**：`api.ts` 加类型化错误（保留端点/状态上下文）、`401` → 提示设置 token、超时、abort。`ws.ts` 加自动重连（指数退避）+ 心跳 + 壳内连接状态灯联动。
- **无障碍**：键盘可达、可见 focus ring、表格/对话框 ARIA、语义 HTML、颜色不作唯一信号（健康/状态都配文字或图标）。
- **测试**：前端 vitest（组件/数据层）+ Playwright（关键流程 e2e：配置 + 测试连接 + remember/recall + 导航高亮）；后端 pytest（新端点 auth/校验/test 行为，真模型调用走 gated 烟测）。

## 8. 分期（地基优先；每期独立 spec→plan→里程碑）

### P2.k — 地基（纯前端，可在 main 做，零 C++，无 migration）
**范围**：设计系统 token + 组件库（手写基础件 + bits-ui 原语）+ 新壳（分组 IA / 当前页高亮 / 三盏健康灯）+ 全 12 面板**套新壳**并接统一 载入/空/错/toast + **DataTable 升级**（排序/筛选/分页/格式化，全表受益）+ 设置页**组件级升级**（show/hide、字段校验、保存 toast、重嵌确认弹窗的前端壳）+ `api.ts`/`ws.ts` 加固 + token 迁入 Access。
**不做**：单面板深度信息重构（P2.m）、多 provider 后端与 `/api/config/test`（P2.l）。设置页本期仍是单一 OpenAI 兼容字段，但 UX 已质变（show/hide、校验、toast）。
**产出**：一个肉眼可见有品味、一致的 app；vitest + Playwright 绿。
**位置**：纯前端 + 极少 Python（无）→ 可在 main 直接做（参考记忆 `python-additive-milestones-run-on-main`：本期不新增 starling.* 子包、不碰 C++，安全）。

### P2.l — 配置·适配器（C++ + Python + TS，worktree + 重建）
**范围**：新 `_core.AnthropicAdapter` + pybind 绑定 + Python provider 工厂（`DashboardEngine._build_chat_adapter`/`_build_embedder`）+ 配置模型 v2（provider 字段 + 预设 + 向后兼容读旧文件）+ `POST /api/config/test` 端点 + 设置 UI（preset 选择器 / 模型下拉 / 测试连接 + 内联结果 / dim 守护 + 重嵌确认 + 进度 / embedder 降级可见 / token 轮换）。
**约束**：API key env-only（含 `ANTHROPIC_API_KEY`）；改 C++/绑定后 `cmake --build` + `cmake --install` 刷新 `_core`；构建走 `scripts/configure_build.py`；**与并行 p2-c 任务协调**（先 pull / 用真 worktree 从最新 main 切）。
**验收**：ctest + pytest + `ci_static_scan.py` 绿；**真模型 QA**——Anthropic 原生 + 至少一个 OpenAI 兼容 provider 端到端跑通（测试连接绿 + remember 抽出 statements）。

### P2.m — 面板深做（前端 + 少量读侧查询）
**范围**：12 面板按组逐组深做（观测 → 交互 → 诊断），每组可再细分为子里程碑。内容见 §6。可能新增/增强读侧查询端点（`queries.py` + routes）。
**约束**：多数纯前端 + 读侧 Python；若新增读端点无 C++/migration 则可在 main 做，否则 worktree。
**验收**：每面板 载入/空/错 态齐 + 无裸 JSON + 关键交互（抽屉/筛选/可视）e2e 覆盖。

## 9. 约束 / 非目标

**约束**：
- API key 仅 env、绝不记录/绑定 Python/提交；secrets 仅在 `~/.starling/starling.json`(0600，gitignored) + 进程内存；`/api/config` 只回 `key_set`；`/api/config/test` 不记录请求体。
- P2.k 纯前端可在 main 做；P2.l 必须 worktree + 重建并与并行任务协调；`cmake --install` 在 C++/绑定改动后必跑。
- 每期真模型 QA（P2.l 配置 + 测试连接尤其）；保留单进程 `run_dashboard.py` + `#token` 登录。
- subscriber/写后代码若涉及须 SAVEPOINT 不 BEGIN（本设计基本不碰写后路径，但若 P2.m 加读侧查询遵循既有约定）。

**非目标（本设计）**：
- 多租户 / 运行时切换 agent·tenant（暂留启动期全局，标注为未来工作）。
- 超出 token 的远程多用户鉴权 / RBAC。
- 移动优先（响应式但桌面为主）。
- Voyage embedder 若非 OpenAI 兼容则不强制实现（标为未来）。

## 10. 成功标准

1. 首跑用户能从**预设**配出可用 LLM + embedder，保存前看到**绿色"测试连接"**，无需翻外部文档。
2. 每个面板都有 载入/空/错 态；无裸 JSON 倒灌；表格可排序/筛选/分页。
3. 12 面板视觉语言一致（token 驱动），明暗双主题等价。
4. **Anthropic 原生 + OpenAI 兼容预设**都端到端跑通（真模型 QA 验）。
5. 配置不可信的三大隐疾消除：测连通可验、dim 不一致有守护、embedder 降级可见。

## 11. 风险与协调

- **并行 p2-c 任务共享 main**：P2.k 纯前端、文件不与其 C++ 改动重叠，可在 main 做（仅 explicit-path `git add`）。P2.l 碰 C++/绑定，必须从最新 main 切真 worktree、独立 venv 重建，合并前 pull + 协调。
- **构建体系已变**：canonical 构建为 `python scripts/configure_build.py --build --test`（local-first 依赖解析）+ `CMakePresets.json`；P2.l 按此构建，不用旧 pip-cmake 手法。
- **Anthropic 适配器是 core 工作**：接口/libcurl/JSON/绑定的确切锚点由 P2.l plan 通过代码探查锁定后再展开 no-placeholder task。
- **真模型 QA 成本**：测试连接与端到端走 gated 烟测（参考 P2.i DashScope、P2.j grounding 端到端做法），不进默认 CI。
